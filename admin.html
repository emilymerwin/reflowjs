A.J.C. multimedia obit tool


Text elements

The following contain only text formatted in markdown and/or HTML. They shouldn't include multimedia elements.

Headline (optional)

This can be anything. It displays very, very large at the top. If there is a background it will have a 1 pixel white outline.

Name

Enter the name as you intend it to be displayed. Use   (non-breaking space) to control where the line break occurs.

Dates

Appear directly under the name. Should be year-of-birth hyphen year-of-death (e.g. 1955 - 2014).

Short bio (optional)

A few lines. Displays under the dates. A bullet list or short paragraph is best. Longer bio material should go in a multimedia block.

Images

These should be URLs to images. Upload them first to [AJC server name]. Please size them appropriately in an image editor.

Mugshot URL

Should be at least 256px wide. Preferably taller than it is wide.

This appears in the primary box along with the name and dates. It is quite possibly the most important thing on the page. Please search [for a photo] or use something from a government or official site or with a creative commons license allowing reuse.


Headline backdrop image URL (optional)

Should be an at least 900px wide. png files preferred.

This will appear full frame no matter how wide the use sets the page. The height will cleverly adjust, but please use a horizontal image. Vertical and square images will likely be too tall at larger screen widths. Exceptions can obviously be made if it looks good that way. The headline is aligned along the bottom of the image.

Instructions for the multimedia blocks

These are the heart of the matter. They can contain maps, images, slideshows, Brightcove video, timelines, embeds and text formatted in Markdown or HTML.

Items appear generally in the order entered, but the automatic layout attempts to put the puzzle together without a lot of awkward white space.

If you're not happy with the layout at one of the widths, tweaking the order definitely makes a difference.

So, what's Markdown?

Markdown is a convenient format for quickly writing HTML without having to write HTML. But, you can also mix and match by including HTML.

For example, text followed by a blank line becomes a <p> element. Lines starting with "* " become a bullet list.

Once you get the hang of it, it's actually quite a bit easier than using a WYSIWYG editor, in part because we've used the flexibility to add shortcuts for common tasks: timelines, maps, Brightcove embeds and slideshows.

Because it's sometimes unfamiliar, the HTML version will appear below the text box you're entering.

AJC shortcuts

Each of following simply starts a new block

  • Google map with info window: Begin a block with "map" (all lower case) one one line, the address to map on a second line and the material for the info window on the third and subsequent lines. The info window can contain basic formatting. Use <br> to add line breaks within the window. 
    • Example:
map
223 Perimeter Center Pkwy, Atlanta, GA 30346 
The offices of the Atlanta Journal-Constitution are lovely and the parking is much better
than it was downtown.
  • Slideshow: Start a block with the word "slideshow" followed by the complete url for the RSS feed of an AJC slideshow. Any text on following lines will be ignored. 
    • Example:
slideshow: http://www.ajc.com/gallery/rss/sports/golf/georgia-golfers-british-open/gCL8d/#5732554
  • Timeline: Start a block with the word "timeline" followed by any headline or setup material, followed by a bullet list of timeline items. Dates can be set on separate bullets or by making them headlines with the bullet. Images can be embedded by using their URL. Other things can be embedded with HTML. Consider whether the material would look better in a wide or narrow box. 
    • Example:
timeline
##A headline
This is some setup text

* 1922
* #March
  Buys a Studebaker
* #May
  Crashes Studebaker
* 1924 - 1929
* http://users.humboldt.edu/ogayle/hist111/BuyBuyBuy.png Works on Wall Street
* 1929
* Stock market crash sets off the Great Depression.

You can also put text after the timeline.
